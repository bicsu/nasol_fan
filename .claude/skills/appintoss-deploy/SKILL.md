# appintoss-deploy

나솔팬즈 앱인토스 배포 수동 작업 진행 가이드 스킬.

## 트리거 조건

다음 요청 시 이 스킬을 사용한다:
- "배포 체크리스트 확인해줘"
- "수동 작업 뭐 남았어"
- "todo 확인해줘"
- "앱인토스 배포 진행"
- "다음 할 거 뭐야"

## 실행 절차

### 1. 현재 상태 파악

최신 todo 파일 읽기:
```
docs/todo/ 디렉토리에서 가장 최근 날짜 파일 읽기
```

### 2. 자동 확인 가능한 항목 체크

코드/파일로 확인 가능한 항목을 직접 점검:

| 항목 | 확인 방법 |
|------|----------|
| `@apps-in-toss/web-framework` 설치 여부 | `package.json` dependencies 확인 |
| Supabase 마이그레이션 파일 존재 여부 | `supabase/migrations/` 파일 목록 확인 |
| `api/` 구조 완성 여부 | `api/` 디렉토리 파일 목록 확인 |
| `.env.example` 항목 누락 여부 | `.env.example` 읽기 |
| `vercel.json` cron 설정 여부 | `vercel.json` 읽기 |

### 3. 수동 작업 상태 질문

자동 확인이 불가한 항목은 사용자에게 직접 질문:

```
다음 항목들 완료했나요? (완료한 번호 말씀해주세요)

1. 토스 콘솔 → 토스 로그인 → Client ID/Secret 확인
2. Vercel 환경변수 추가 (TOSS_CLIENT_ID, TOSS_CLIENT_SECRET 등)
3. Supabase 마이그레이션 006, 007 적용
4. Supabase Auth Redirect URL 설정
5. 앱 번들 빌드 (.ait 파일 생성)
6. 앱인토스 콘솔 업로드 + QR 테스트
```

### 4. 완료 항목 업데이트

사용자가 완료 항목을 알려주면:
- `docs/todo/YYYYMMDD.md` 에서 해당 항목 `- [ ]` → `- [x]` 로 업데이트
- 다음 할 작업을 명확히 안내

### 5. 막힌 항목 지원

사용자가 특정 단계에서 막혔다고 하면:
- 해당 단계의 구체적인 가이드 제공
- 필요 시 코드 수정 (예: 환경변수명 불일치, API 오류 등)
- `appintoss-qa` 스킬과 연계하여 코드 정합성 재검증

## 출력 형식

```
## 현재 진행 상황

완료: X / 전체: Y

### 완료
- [x] ...

### 다음 할 것
1. [가장 급한 항목] — [간단한 방법]

### 막힌 게 있으면 말씀해주세요
```

## 연계 스킬

- `appintoss-qa`: 코드 품질/컴플라이언스 재검증 필요 시
- `nasol-build`: 추가 코드 구현 필요 시
